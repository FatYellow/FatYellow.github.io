什么是hexo

hexo是一款基于github的博客框架，相比与Wordpress，它的优势在于不需要购买额外的服务器，缺点也很明显，git真的可以调到人心态崩溃。hexo本身原生支持markdown语言和EJS语言(我也不知道这是个什么玩意)，只需要更改自动生成的网页源文件的拓展名就可以

配置Github仓库

注册Github账户

这都不会的话建议remake

新建仓库

新建一个名为<username>.github.io的仓库，此处<username>为你的用户名，否则会影响GitHub将域名连接到你的仓库上，将来你的网站域名就将是http://<username>.github.io。
由此可见，每一个GitHub账户最多只能拥有一个可以直接使用域名访问的仓库。

注意事项

  • 注册时一定要验证邮箱
  • 仓库创建不一定会立刻生效，下文中对仓库的更新同理

    配置SSH key

    SSHkey是用于绑定你的电脑上的git与你的github账户的，如果不绑定则无法将你的本地更新推送到仓库上。 如果提示：No such file or directory则说明本机并未进行过ssh密钥的生成配对 然后连续三次输入回车，最终会在该用户目录下生成.ssh\id_rsa.pub文件。使用记事本打开并复制文件中的内容，打开你的GitHub主页，进入个人设置 –>SSH and GPG keys –> New SSH key，将刚才复制的内容粘贴到Key框内，Title框可以随便填写，可以填一些便于你进行Key管理的标题。

    测试是否绑定成功

    如果提示Are you sure you want to continue connecting (yes/no)?并输入yes，你就会看到
    Hi FatYellow! You've successfully authenticated, but GitHub does not provide shell access.
    到这里就说明你的SSH已经配置成功了。
